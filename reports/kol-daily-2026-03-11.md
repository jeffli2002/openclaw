## 🧠 深度观点

### 开发者实战：AI编程代理的信任难题

**文章来源**：Claude Code Camp - "I'm Building Agents That Run While I Sleep"

**核心洞察**：

1. **问题**：AI写的代码越来越多，但如何验证正确性？
   - 团队使用Claude后每周合并40-50个PR（之前10个）
   - 代码审查时间成本激增

2. **自检陷阱**：AI写测试检查AI写的代码 = 自我庆祝机器
   - 测试证明代码做了"AI认为你要的事"
   - 不是"你实际要的事"

3. **解决方案：TDD + 验收标准**
   - 先写验收标准（AC），再让AI构建
   - 用Playwright浏览器代理验证每个AC
   - 失败才需要人工审查

**实践框架**：
```
AC-1: 成功登录 → 重定向到/dashboard
AC-2: 错误密码 → 显示"Invalid email or password"
AC-3: 空字段验证 → 提交按钮禁用
AC-4: 限流 → 5次失败后阻止60秒
```

**🔥 KOL辣评**：与其相信AI不会犯错，不如告诉AI什么才算"对"。验收标准是AI时代的TDD。
