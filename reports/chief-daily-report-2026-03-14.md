## 💡 问题和反思

### 1. 凌晨 3 个 error 的根因，其实都是“配置节拍”问题
- **问题描述**：`daily-content-publish`、`ai-kol-daily-newsletter`、`sync-github-18-00` 凌晨都处于 error。
- **影响范围**：直接影响内容节奏、老板日更感知、仓库备份稳定性。
- **根本原因**：timeout 设计过紧；KOL 交付配置不完整。
- **改进措施**：今天已经完成超时放宽与投递补齐，后续要把这套检查前置到 cron 上线前。

### 2. 飞书多维表格链路暴露出“看起来成功，实际数据不干净”的问题
- **问题描述**：出现 **10 条空记录** 和错误年份时间戳。
- **影响范围**：影响文档归档可信度，容易让后续分析基于脏数据。
- **根本原因**：Bitable placeholder rows 机制未预处理；时间戳计算复用了历史常量；写后未抽样校验。
- **改进措施**：创建后先清空 placeholder、写前校验年份范围、写后抽样核查。

### 3. 飞书权限 API 仍是今天唯一未真正闭环的技术债
- **问题描述**：给文档/应用加协作者时仍报 validation 错误。
- **影响范围**：会影响后续文档协作自动化程度。
- **根本原因**：飞书权限 API 对应用协作者的 `member_type/type` 组合要求更特殊。
- **改进措施**：短期先以 UI 手动授权兜底；中期把 API 参数矩阵单独验证透。
