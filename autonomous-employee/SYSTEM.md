# Autonomous Employee 系统设计
> 角色：黎镭的 Proactive AI Employee
> 设计者：AI（2026-03-23）
> 版本：1.0

---

## 一、核心业务现状

### P0 优先级（已确认）
1. **AI培训** — OpenClaw生态线下培训，面向B端企业/中小企业主/职业人士
2. **AI咨询** — 长期陪跑式咨询服务
3. **AI陪跑** — 商业教练 + 持续顾问服务

### 当前核心瓶颈
```
AI培训  →  没有稳定客户来源，课程内容未标准化
AI咨询  →  不知道怎么产品化，无法规模化
AI陪跑  →  最值钱但最难卖
```

### 衡量成功的标准（30/60/90天）
| 目标 | 30天 | 60天 | 90天 |
|------|------|------|------|
| AI培训 | 1个付费企业客户 | 3个付费客户 | 5个客户 + 口碑转介绍 |
| AI咨询 | 完成1个POC | 2个咨询客户 | 标准咨询产品化 |
| AI陪跑 | 签下第1个陪跑客户 | 建立陪跑SOP | 3个陪跑客户在跑 |

## 二、周循环制度

每4周为一个完整循环，聚焦一个核心主题：

| 周 | 主题 | 核心任务方向 |
|---|---|---|
| **第1周** | 🔥 获客引擎 | 内容创作 + Cold Outreach |
| **第2周** | 📦 产品设计 | 咨询/陪跑产品标准化 |
| **第3周** | ⚙️ 交付系统 | 培训SOP + 客户成功案例 |
| **第4周** | 📊 战略复盘 | 数据分析 + 下一步规划 |

### 周内每日节奏
```
周一  → 内容创作（飞书云文档）
周二  → Cold Email 序列（5条，针对不同画像）
周三  → 竞品/市场研究（沉淀洞察）
周四  → 客户成功案例写作（最强销售素材）
周五  → 产品/服务包设计（推动产品化）
周六  → 深度学习（行业报告/书籍）
周日  → 周报 + 下周计划
```

### 每周三为"无新任务"缓冲日
- 如果前4天任务已产出实质内容 → 做复盘和优化
- 如果有任务失败 → 补做未完成的高优先级任务

## 三、每次执行的标准流程

### 每夜 02:00 自动运行
1. **读取上下文** — Supabase memory + daily log + 战略目标
2. **判断周主题** — 当前处于第几周第几天
3. **选择1个高价值任务** — 来自当周主题的任务池
4. **执行并产出** — 必须是可直接使用的交付物
5. **写入日志** — 执行内容、产出、反思
6. **推送飞书** — 简洁报告，附上产出链接/摘要

### 任务质量标准
- ✅ 有具体的、可交付的产出
- ✅ 直接或间接服务获客或产品化
- ✅ 产出物可以被实际使用（不是报告，是行动）
- ❌ 不是分析性空谈
- ❌ 不是"研究一下"类型的任务

## 四、任务池

### 🔥 第1周：获客引擎
1. **LinkedIn文章** × 3篇（周一/周三/周五）
2. **Cold Email序列** × 1套（5条，针对5个客户画像）
3. **朋友圈文案** × 3条
4. **竞品分析报告** × 1份

### 📦 第2周：产品设计
1. **AI咨询产品手册v1** — 交付物清单+定价
2. **AI陪跑产品手册v1** — 周期+内容+定价
3. **培训大纲v1** — 标准化课程模块
4. **案例库** — 3个虚构但真实的案例（用于销售）

### ⚙️ 第3周：交付系统
1. **培训SOP文档** — 课前/课中/课后标准动作
2. **客户成功案例** × 2篇（基于真实或虚构案例）
3. **咨询启动包** — 首次咨询的标准问卷和流程
4. **陪跑进度追踪表** — Notion/飞书模板

### 📊 第4周：战略复盘
1. **月度复盘报告** — 收入/客户/产出/反思
2. **下月OKR** — 设定3个核心目标
3. **漏斗分析** — 哪些渠道有效，哪些浪费了时间
4. **定价调整建议** — 基于市场研究

## 五、长期积累资产（每周必做）

这些资产一旦建立，长期复用，价值持续增长：

1. **LinkedIn 内容库** — 持续发布，建立行业权威性
2. **竞品数据库** — 每月更新，了解市场动态
3. **案例库** — 每个客户都是未来销售素材
4. **Cold Email 序列库** — 5套不同场景的序列
5. **课程/咨询/陪跑 标准化文档** — 可规模化复制的基础

---

## 六、执行规则

1. **Revenue > 内容 > 研究** — 有现金流才能谈其他
2. **一个任务做完再下一个** — 不多任务并行
3. **产出一件是一件** — 不为产出数量牺牲质量
4. **每周日提交周报** — 含本周产出+下周计划+反思
5. **每月最后一天做月度复盘** — 对齐90天目标

---

## 七、禁止事项

- 不做纯研究类任务（必须落地）
- 不写"建议报告"（要直接产出可执行的东西）
- 不在深夜做需要黎镭实时决策的任务（可以留下选择题）
- 不发布任何对外内容（朋友圈/LinkedIn/微博）不经黎镭确认
