# 公司规则

> 最后更新: 2026-02-26

---

## 核心原则

### 1. 只有1个真正运行的Agent
- 通过system prompt + memory namespace切换角色
- Chief Agent负责任务调度和角色分配
- 禁止自动多线程
- Product和Coding Agent禁止并行执行

### 2. Memory管理规则
- **只有Chief可以写** chief/ 和 projects/ 目录
- Sub-Agent只读必要上下文，不存长期记忆
- 所有报告append-only，禁止rewrite
- agents// 仅用于执行缓存，Chief不读取原始内容

### 3. 任务执行规则
- 单Agent执行模式
- 互斥对：coding_agent ↔ product_agent
- 执行前检查当前Agent状态
- 完成后记录状态

### 4. 沟通规则
- 大方向确认：涉及重大策略/方向不确定时，主动寻求确认
- 小事自己扛：执行层面的问题自己搞定

---

## 质量标准

### 输出规范
- 数据优先：能用数字表达的就不用文字
- 描述性语言：避免简单罗列，要叙述性的描述
- 项目视角：站在整体项目角度

### 禁止事项
- 禁止使用"一些"、"若干"、"部分"等模糊词
- 禁止简单罗列任务清单
- 禁止缺少数据的概括性描述

---

## 决策记录

### 2026-02-26: Memory架构升级
从"每个Agent独立memory"改为"集中式记忆 + 分布式执行"

**原因**：
- 旧架构导致记忆碎片化
- 多Agent同时写入造成冲突
- 难以追踪战略决策

**新架构优势**：
- 记忆集中管理，清晰可追溯
- 只有Chief有写权限，避免冲突
- 分层设计，执行与战略分离
