# 通用知识 - 所有Agent共享

## 飞书文档操作规范

### 写入飞书云文档
- 使用 `feishu_doc` 工具的 `write` 或 `append` action
- 写入前先检查文档是否存在
- 重要内容建议使用 `append` 而非覆盖
- 写入后验证返回结果

### 写入飞书Bitable
- 使用 `feishu_bitable_create_record` 创建记录
- 字段格式：
  - 文本: "string"
  - 数字: 123
  - 单选: "选项名"
  - 多选: ["选项1", "选项2"]
  - 日期时间: 时间戳(毫秒)
  - 用户: [{id:'ou_xxx'}]

### 常见错误处理
- PGRST205: 表不存在 → 检查表名和schema
- 权限不足 → 检查app_scopes

## API调用规范
- 调用外部API后必须验证返回
- 失败时提供备选方案
- 记录token使用量

## 通用原则
- 保持简洁，避免冗余
- 用数据说话
- 结论先行

## ❌ 经验教训：Cron任务发送目标配置

### 错误描述
配置Cron任务的`delivery`时，错误地将所有任务的发送目标设为用户DM，而不是对应的子群ID。

### 错误配置示例
```json
{
  "delivery": {
    "mode": "none",
    "to": "ou_xxx"  // 错误：应该是子群ID
  }
}
```

### 正确配置
| Agent | 发送目标 |
|-------|----------|
| Chief日报 | 用户DM: `ou_aeb3984fc66ae7c78e396255f7c7a11b` |
| Content | Content子群: `oc_xxx` |
| Growth | Growth子群: `oc_xxx` |
| Product | Product子群: `oc_xxx` |
| Coding | Coding子群: `oc_xxx` |

### 子群ID列表
- Content: `oc_1e781764ad5c3b463eef7d0aee1de2a9`
- Coding: `oc_3eca5aac26f0a945e0b4febc76214066`
- Product: `oc_76d55844d04e400ed71327069580be96`
- Growth: `oc_86babca945b808774c67a3ef130f64a5`
- Finance: `oc_e810980541f92c802b8e970f49854381`

### 教训
- 配置Cron任务的delivery时，必须指定正确的发送目标
- Agent专属任务 → 发到对应子群
- 全局汇总任务 → 发到用户DM
