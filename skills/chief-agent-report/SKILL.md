# Chief Agent 每日汇报

## 触发条件

当需要生成Chief Agent日报、每周汇报、项目进度汇报时使用此skill。

## 核心原则

**数据优先**：能用数字表达的就不用文字
**描述性语言**：避免简单罗列，要叙述性的描述
**项目视角**：站在整体项目角度，而非单个Agent角度

---

## 输出格式要求

### 整体结构

1. **📊 今日工作完成情况**
2. **🚀 进行中的项目**
3. **📝 今日工作总结**
4. **💡 问题和反思**
5. **✨ 创造性建议**
6. **📅 明日规划**
7. **🔮 本周整体规划**（周维度视角）

---

## 详细要求

### 1. 今日工作完成情况

**必须包含**：
- **完成数量**：如"完成了5个任务"、"处理了12个请求"
- **完成率/成功率**：如"完成率95%"、"转化率80%"
- **时间跨度**：如"从上午9点持续到下午6点"
- **关键里程碑**：按时间线描述每个关键节点

**描述方式**：
```
❌ 完成了内容生成任务
✅ 通过Content Factory完成了3篇公众号文章创作，平均耗时45分钟/篇，总计产出超过15000字，其中AI行业解读获得最高阅读量
```

### 2. 进行中的项目

**必须包含**：
- **项目名称**：明确的命名
- **当前阶段**：需求分析/开发中/测试中/部署中
- **已完成百分比**：如"已完成65%"
- **剩余工作量**：明确还有多少未完成
- **预期完成时间**：具体时间或里程碑

**描述方式**：
```
❌ 官网重构项目在进行中
✅ 官网重构项目当前处于UI开发阶段（进度65%），已完成首页、博客列表页、详情页的响应式布局，正在开发产品展示页面，预计本周五完成全部页面开发，下周一进入测试阶段
```

### 3. 今日工作总结

**必须包含**：
- **核心成果**：最重要的3-5项，用动词开头描述
- **关键决策**：今日做了哪些重要决策，背景和原因
- **跨Agent协作**：哪些Agent之间进行了协作，产出是什么

**描述方式**：
```
❌ 完成了多个任务
✅ 今日核心成果包括：(1)内容团队通过content-factory产出3篇高质量公众号文章，其中AI工具推荐类文章已发布即获得500+阅读；(2)开发团队完成官网首页的性能优化，首屏加载时间从2.3s降至1.1s，提升52%；(3)决策将下周工作重心从功能开发转向用户增长
```

### 4. 问题和反思

**必须包含**：
- **问题描述**：具体发生了什么
- **影响范围**：影响多大
- **根本原因**：为什么会发生
- **改进措施**：如何避免再次发生

**描述方式**：
```
❌ 遇到一些技术问题
✅ 今日遇到的问题是：飞书文档写入出现2次空内容问题，影响日报发送。通过排查发现是API调用时未正确处理异步返回，导致数据未写入就已关闭连接。改进措施：增加写入确认机制，在每次写入后增加验证查询
```

### 5. 创造性建议

**必须包含**：
- **具体方案**：可执行的建议
- **预期收益**：量化预期收益
- **实施路径**：如何落地

**描述方式**：
```
❌ 可以优化工作流程
✅ 建议引入自动化测试覆盖：当前代码发布后需手动测试，建议增加单元测试和集成测试，目标是将测试覆盖率从30%提升至70%，预计可减少每次发布后30分钟的回归测试时间，使发布频率从每周1次提升至每周2-3次
```

### 6. 明日规划

**必须包含**：
- **具体任务**：明确要做什么
- **优先级**：P0/P1/P2
- **预期产出**：可量化的结果
- **时间安排**：大致时间分配

**描述方式**：
```
❌ 完成一些任务
✅ 明日规划：(1)P0-完成产品页面开发，预计产出5个4小时；(2)P0-公众号文章发布，页面，耗时2篇，耗时2小时；(3)P1-用户反馈收集，目标是收集20份有效反馈，耗时1小时；(4)P2-代码review，耗时1小时
```

### 7. 本周整体规划（周维度）

**必须包含**：
- **本周核心目标**：1-2个核心目标
- **关键里程碑**：每天的里程碑
- **完成标准**：什么算完成
- **风险预警**：可能的风险及应对
- **资源需求**：需要的人和资源

**描述方式**：
```
本周核心目标：完成官网重构上线（目标：周三前）、公众号粉丝增长15%（当前2500→2875）

关键里程碑：
- 周一：完成剩余3个页面开发
- 周二：完成全站测试，修复bug
- 周三：上线新官网
- 周四：收集用户反馈
- 周五：优化迭代

风险预警：
1. 官网重构可能遇到兼容性问题，已准备IE11降级方案
2. 内容发布受平台审核影响，已准备3篇备用文章

资源需求：
- 需要1名开发协助测试
- 需要设计支持1小时确认UI细节
```

## 数据获取

在生成汇报前，必须执行：

1. **查询各Agent状态**：`subagents list` 查看所有活跃Agent
2. **获取项目上下文**：读取相关项目文件了解当前项目状态
3. **查看历史记录**：查看之前的汇报记录确保连贯性
4. **获取关键数据**：如有可能，获取具体的项目指标

---

## 禁止事项

- ❌ 禁止使用"一些"、"若干"、"部分"等模糊词
- ❌ 禁止简单罗列任务清单
- ❌ 禁止缺少数据的概括性描述
- ❌ 禁止与上周/昨日无关联的独立汇报

---

## 输出检查清单

生成汇报后，自检以下内容：

- [ ] 今日工作完成情况是否有具体数字？
- [ ] 进行中的项目是否有百分比？
- [ ] 核心成果是否用描述性语言？
- [ ] 问题是否有影响范围和根因分析？
- [ ] 建议是否有预期收益量化？
- [ ] 明日规划是否标明优先级？
- [ ] 本周规划是否与项目整体目标关联？
