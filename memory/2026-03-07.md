## 00:43 小红书登录/发布链路 durable memory
- Feishu 发图片问题已定位：当图片源在 `/tmp/...` 等非允许目录时，OpenClaw/Feishu 发送会退化为发送“路径文本”；将图片复制到 workspace 内并用 `message(media=...)` 发送后，二维码图片可正常投递到 Feishu。

- 通过用户提供的小红书 cookies（核心至少含 `a1`、`web_session`，并配合 creator 相关 token）注入远端 Chrome 的 CDP `Network.setCookies` 后，已能直接进入 `https://creator.xiaohongshu.com/publish/publish?source=official`，且可见“上传图文 / 写长文 / 上传区域”。

- 当前可复用的小红书 cookies 已备份到 `/root/.openclaw/credentials/xiaohongshu.json`，供后续 creator 发布恢复与图文发布半链路测试使用。
