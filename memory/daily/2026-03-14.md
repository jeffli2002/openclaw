## 11:30 - 飞书多维表格创建复盘

### 任务背景
将之前创建的飞书云文档归档到多维表格，包含标题、链接、创建时间。

### 错误1：空行问题
**现象**：创建多维表格后出现10行空记录
**原因**：`feishu_bitable_create_app` 创建应用时会自动生成 placeholder rows
**解决方案**：通过 API 逐条删除空记录
**经验**：
- 创建 Bitable 后先用 `feishu_bitable_list_records` 检查是否有空记录
- 如有空记录，立即用 delete API 逐条删除
- 或使用 `cleaned_placeholder_rows: true` 参数（如果支持）

### 错误2：时间戳错误
**现象**：创建时间出现 2025 年日期（如3月18日），实际应为2026年
**原因**：计算时间戳时错误地使用了2025年基准（1458xxx），应为2026年（1773xxx）
**原因分析**：
- 文档标题含"2026-03-10"等日期，我误以为只需提取月日
- 错误地复用了历史代码中的时间戳常量
- 没有校验计算结果是否在合理范围内
**解决方案**：根据文档标题中的日期，重新计算2026年正确的时间戳并更新
**经验**：
- 时间戳必须根据当前年份重新计算，不能复用历史值
- 写入前校验时间戳范围（2026年应为 177xxx 开始）
- 重要数据写入后应抽样验证

### 错误3：API 权限问题（未解决）
**现象**：尝试用 API 添加文档协作者失败，权限 API 一直报 validation 错误
**原因**：飞书权限 API 对应用（app）类型的协作者支持特殊，需要正确的 member_type 和 type 组合
**经验**：
- 飞书 API 中应用协作者权限需要 `type=docx` 作为 query param
- 复杂权限场景优先使用 UI 手动处理

### 教训总结

## 23:26 - 飞书语音 Emoji 过滤修复

### 问题
- 用户反馈：TTS 语音播放时不应朗读 emoji，否则会有杂音
- 原因：edge-t 会把 emoji 字符当作异常文本处理，导致生成音频极短（1.59秒）

### 修复方案
- 在 `build_feishu_voice.py` 中添加 `remove_emoji()` 函数
- TTS 合成前过滤掉 emoji，保留原文显示
- Emoji 正则只匹配 emoji 字符范围，不触及其他 Unicode 字符

### 代码变更
- 文件：`/root/.openclaw/workspace-content/skills/scripts/build_feishu_voice.py`
- 新增：`remove_emoji()` 函数
- 逻辑：合成前过滤 emoji，输出不受影响

### 经验
- emoji 字符会导致 edge-t 异常处理，生成音频损坏
- 解决方案：分离"显示文本"和"朗读文本"，朗读时过滤 emoji
- regex 需精确范围，避免误删 CJK 中文字符
