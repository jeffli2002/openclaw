# Memory Flush — 2026-03-08

## Durable learnings

### 1. 2nd Brain deployment hardening
- Vercel / Next.js builds in multi-workspace or multi-lockfile environments are more stable when `next.config.ts` explicitly sets:
  - `outputFileTracingRoot: process.cwd()`
- Local verification path used successfully:
  - `npm ci`
  - `npm run build`
- This should be treated as a standard fix when Vercel reports root / framework detection ambiguity around the 2nd Brain project.

### 2. 2nd Brain search robustness
- Search UI can crash if frontend code assumes all searchable fields are non-null and calls `.toLowerCase()` directly.
- Durable fix pattern:
  - centralize a null-safe text normalizer
  - route all search matching through a safe helper
  - include optional fields like `content`, `path`, `schedule`, `status` through that helper only

### 3. Agents token chart fallback pattern
- Agents / token charts should not depend only on local cron run files.
- In cloud deployments (for example Vercel), local cron history may be unavailable.
- Durable fallback pattern:
  - primary source: local cron/runs trend data
  - fallback source: Supabase task snapshot data
  - UI fallback: always render a current token distribution chart even when historical trend is missing

### 4. Cron `summary_only` delivery rule
- For cron jobs whose policy is effectively `summary_only`, default delivery should be `none`, not announce-to-user.
- If such a job truly needs to notify the boss, the agent must use `message` with explicit routing:
  - `channel=feishu`
  - `target=user:ou_aeb3984fc66ae7c78e396255f7c7a11b`
- Do not rely on isolated cron session context for outbound user messaging.
- This specifically fixed repeated `⚠️ ✉️ Message failed` behavior on `product-competitor-analysis`.

### 5. Product cron repair pattern
- `product-competitor-analysis` became healthy after:
  - ensuring required product workspace memory files/directories exist
  - aligning job delivery mode with `summary_only`
  - updating job prompt so boss notifications require explicit `channel` and `target`
- This is the standard repair checklist for isolated agent cron tasks that fail only when trying to notify.
